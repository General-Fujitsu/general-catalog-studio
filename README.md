# GENERAL Catalog Studio 2026/27 – Version 3

Statische Einzeldatei für GitHub Pages. Der komplette Katalogdatensatz ist direkt in `index.html` eingebettet. Es gibt keine Datenbank und keine Projektspeicherung.

## Neue Funktionen in Version 3

- verbesserte Analyse von Kundenmails und Fließtexten
- mehrere Räume werden auch bei unterschiedlichen Schreibweisen zuverlässig erkannt
- Unterstützung für Angaben wie `Raumgröße`, `Raumgröse`, `m²`, Raumhöhe, Rohrlänge und explizite kW-Angaben
- individuelle Auswahl des Innengeräts je Raum
- automatische Multi-Split-Prüfung anhand von:
  - Anzahl der Innengeräte
  - Geräte-Index
  - angeschlossener Kühlleistung
- passendes Duo-, Triple-, Quadro- oder Penta-Außengerät wird vorgeschlagen
- alternative passende Außengeräte können direkt gewählt werden
- im Warenkorb wird bei Multi-Split-Innengeräten automatisch auf ein fehlendes oder unpassendes Außengerät hingewiesen
- Zubehör wird nicht mehr pauschal hinzugefügt
- Kondensatpumpen, Kältemittelleitungen, Dämmsockel, Umweltschutzwanne, Reparaturschalter und Winterregelung können einzeln ausgewählt und in der Menge angepasst werden
- verbesserte Assistentenstruktur mit drei klaren Arbeitsschritten

## Weitere Funktionen

- Suche nach Artikelnummer, Modell, Leistung, Bauform, Farbe und Schlagworten
- Produktdetails mit Katalogseite und Katalogauszug
- Artikelliste und Team-Stuttgart-Mailtext
- lokale PDF-, LV- und Auslegungsanalyse im Browser
- Export als TXT und CSV

## GitHub Pages aktualisieren

1. ZIP-Datei auf dem Computer entpacken.
2. Im Repository **Add file → Upload files** öffnen.
3. `index.html`, `README.md` und `.nojekyll` hochladen.
4. Die vorhandenen Dateien ersetzen beziehungsweise überschreiben.
5. Unten **Commit changes** auswählen.
6. Unter **Actions** warten, bis die GitHub-Pages-Bereitstellung grün ist.
7. Die veröffentlichte Seite mit `Strg + F5` vollständig neu laden.

Die ZIP-Datei selbst nicht in das Repository hochladen.

## Datenschutz und Zugriff

Die Anwendung überträgt Projekttexte und eingelesene Dateien nicht an einen Server. PDF-Dateien werden mit PDF.js im Browser verarbeitet. Der in `index.html` eingebettete Katalog ist jedoch für jeden sichtbar, der Zugriff auf die GitHub-Pages-Adresse oder das Repository hat.
