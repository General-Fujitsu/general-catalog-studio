# GENERAL Catalog Studio 2026/27 – Version 4

Statische Browser-Anwendung für GitHub Pages. Der Katalogdatensatz ist direkt in `index.html` eingebettet. Es gibt keine Datenbank und keine Projektspeicherung.

## Katalogstruktur

Die Navigation folgt dem Aufbau des GENERAL-Preiskatalogs 2026/2027:

1. Single-Split
2. Multi-Split – Duo, Triple, Quadro und Penta
3. Simultan-Multi – eigener Bereich für baugleiche Innengeräte im Gleichlauf
4. VRF-Systeme
5. DX-Systeme
6. Zubehör und CCM

### VRF-Auswahl

Im VRF-Bereich wird schrittweise ausgewählt:

1. Kältemittel R32 oder R410A
2. Komponente, beispielsweise Außen- oder Innengerät
3. Gerätetyp beziehungsweise VRF-Serie

### Simultan-Multi

Simultan-Multi wird getrennt von Duo bis Penta behandelt. Bei einer im Katalog freigegebenen Kombination werden das passende Außengerät und der erforderliche Kältemittelverteiler einzeln vorgeschlagen. Voraussetzung sind baugleiche Innengeräte gleicher Art und Leistung.

## Überarbeitete Produktdaten

- Außen- und Innengeräte werden über Modellpräfixe, Katalogseiten und geprüfte Artikelzuordnungen getrennt.
- Bekannte fehlerhafte Zuordnungen aus der automatischen Katalogauslesung wurden korrigiert.
- Produktkarten zeigen den Katalogbereich, die Komponente, den Gerätetyp und das Kältemittel.
- Für wichtige Gerätefamilien werden offizielle GENERAL-Produktbilder verwendet. Falls ein Bild nicht geladen werden kann, erscheint automatisch eine integrierte Gerätegrafik.

## Zubehör

Zubehör wird nicht automatisch in den Warenkorb gelegt. Im Zubehör-Assistenten können Position und Menge einzeln gewählt werden, unter anderem:

- Kondensatpumpen
- Kältemittelleitungen
- Dämmsockel und Umweltschutzwannen
- Reparaturschalter
- Busleitungen
- VRF-Verteiler und Sicherheitszubehör

## Weitere Funktionen

- Suche nach Artikelnummer, Modell, Leistung, Bauform, Farbe und Schlagworten
- Produktdetails mit Katalogseite und Katalogauszug
- Analyse von Kundenmails und Fließtexten mit mehreren Räumen
- individuelle Innengeräteauswahl je Raum
- Multi-Split-Prüfung nach Anzahl, Geräteindex und Anschlussleistung
- lokale PDF-, LV- und Auslegungsanalyse im Browser
- Artikelliste und vorbereiteter Team-Stuttgart-Mailtext
- Export als TXT und CSV

## GitHub Pages aktualisieren

1. ZIP-Datei auf dem Computer entpacken.
2. Im Repository **Add file → Upload files** öffnen.
3. `index.html`, `README.md` und `.nojekyll` hochladen.
4. Vorhandene Dateien ersetzen und **Commit changes** auswählen.
5. Unter **Actions** warten, bis die GitHub-Pages-Bereitstellung grün ist.
6. Die veröffentlichte Seite mit **Strg + F5** vollständig neu laden.

Die ZIP-Datei selbst nicht in das Repository hochladen.

## Datenschutz und Zugriff

Projekttexte werden nicht an einen eigenen Server übertragen. PDF-Dateien werden lokal im Browser verarbeitet. Die Anwendung lädt einzelne offizielle Produktbilder von der GENERAL-Webseite nach. Der eingebettete Katalog ist für alle Personen sichtbar, die Zugriff auf die GitHub-Pages-Adresse oder das Repository haben.
